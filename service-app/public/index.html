Applications

NAME
       goli applications - Manage Cloud Foundry applications

SYNOPSIS
       goli applications [OPTIONS] [APPLICATION_NAME]
       goli a [OPTIONS] [APPLICATION_NAME]

DESCRIPTION
       The goli applications command, or its alias goli a, is used to manage Cloud Foundry
       applications.
       It allows listing applications, retrieving details, modifying environment variables,
       managing instances, and performing actions such as restarting, restaging, or enabling SSH access.

OPTIONS
       -h, --help
              Display this help message and exit.

ARGUMENTS
       APPLICATION_NAME
              The name of the specific application to perform actions on. This argument
              is required when managing a particular application.
              If not specified, the command lists all applications in the current space.

ACTIONS
       The goli applications command supports several actions for managing applications:

       change-env
              Modify the environment variables of the application.

       create-env
              Add a new environment variable to the application.

       details
              Display the current details and health of an application.

       enable-ssh
              Enable SSH access for the application.

       envs
              View the environment variables of the application.

       logs
              View the logs of the application.

       manipulate-instances
              Perform actions on the application instances.

       postgres
              Create an SSH tunnel to the PostgreSQL instance bound to the application.

       redis
              Create an SSH tunnel to the Redis instance bound to the application.

       restage
              Restage the application to apply any configuration changes.

       restart
              Restart the application.

       show-bound-instances
              Display the instances currently bound to the application.

INTERACTIVE MODE
       The goli applications command also supports an interactive mode.
       When run without a specific application name or action, the command provides a menu
       of available options for selection.

EXAMPLES
       List all applications in the current Cloud Foundry space:
              goli applications

       Target a specific application and perform actions:
              goli applications my-app

       View logs for a specific application:
              goli applications logs my-app

       View details and health status of an application:
              goli applications details my-app

       Add an environment variable to an application:
              goli applications create-env my-app --name NEW_VAR --value new-value

       Restart a specific application:
              goli applications restart my-app

change-env

NAME
       goli applications change-env - Modify an existing environment variable for an application

SYNOPSIS
       goli applications change-env APPLICATION_NAME [--name ENV_VAR_NAME] [--value ENV_VAR_VALUE]
       goli a change-env APPLICATION_NAME [--name ENV_VAR_NAME] [--value ENV_VAR_VALUE]

DESCRIPTION
       The change-env command allows modifying the value of an existing environment 
       variable for a specified application.
       If the environment variable does not exist, the command will fail.

       If --name or --value is not specified, the command enters interactive mode, 
       allowing the user to choose an environment variable and input a new value.

OPTIONS
       -h, --help
              Display this help message and exit.

       --name <ENV_VAR_NAME>
              (Optional) The name of the environment variable to modify.
              If not specified, the command enters interactive mode.

       --value <ENV_VAR_VALUE>
              (Optional) The new value of the environment variable.
              If not specified, the command enters interactive mode.

ARGUMENTS
       APPLICATION_NAME
              The name of the application whose environment variable should be modified.
              This argument is required.

EXAMPLES
       Modify an environment variable with explicit name and value:
              goli applications change-env my-app --name LOG_LEVEL --value debug

       Run in interactive mode (select an environment variable and input a value):
              goli applications change-env my-app

SEE ALSO
       create-env, envs

restage

NAME
       goli applications restage - Restage a specified application

SYNOPSIS
       goli applications restage APPLICATION_NAME [OPTIONS]
       goli a restage APPLICATION_NAME [OPTIONS]

DESCRIPTION
       The restage command is used to restage an application in Cloud Foundry.
       Restaging an application means that it will be re-deployed without changing
       the application's codebase.
       It is particularly useful when environmental changes potentially alter
       the application's runtime behavior, even though the application code itself remains unchanged.

       This helps ensure that the application picks up any changes to its environment
       without requiring a full redeployment.

OPTIONS
       -h, --help
              Display this help message and exit.

       -r, --rolling
              Perform a rolling restage, restarting application instances incrementally.
              This strategy ensures that some instances are kept running while others
              are restaged, reducing downtime.

EXAMPLES
       Restage the "my-app" application:
              goli applications restage my-app

       Restage the "my-app" application with rolling restage:
              goli applications restage my-app --rolling

       Use alias:
              goli a restage my-app

       Use alias with rolling restage:
              goli a restage my-app --rolling

SEE ALSO
       restart

Instances

NAME
       goli instances - View or interact with service instances bound to applications

SYNOPSIS
       goli instances [OPTIONS] [INSTANCE_NAME]
       goli i [OPTIONS] [INSTANCE_NAME]

DESCRIPTION
       The goli instances command, or its alias goli i, is used to interact with service instances
       bound to applications.
       It allows managing service instance credentials, generating keys, viewing capabilities,
       and unbinding services from applications.

OPTIONS
       -h, --help
              Display this help message and exit.

ARGUMENTS
       INSTANCE_NAME
              The name of the specific service instance to perform actions on. This argument
              is required when managing a particular instance.
              If not specified, the command lists all service instances in the current space.

ACTIONS
       The goli instances command supports several actions for managing service instances:

       capabilities
              View the capabilities exposed by the service instance.

       create-key
              Generate a new key for the service instance, enabling secure access.

       generate-client-token
              Generate a client token for accessing the service instance.

       show-bound-apps
              List the applications currently bound to the service instance.

       show-credentials
              View the credentials for a specific service instance.

       unbind
              Unbind the service instance from a specific Cloud Foundry application.

INTERACTIVE MODE
       The goli instances command also supports an interactive mode.
       When run without a specific instance name or action, the command provides a menu
       of available options for selection.

EXAMPLES
       List all service instances in the current Cloud Foundry space:
              goli instances

       Target a specific service instance and perform actions:
              goli instances my-db

       View capabilities for a specific instance:
              goli instances capabilities my-db

       Generate a new key for a service instance:
              goli instances create-key my-db

       Generate a client token for a service instance:
              goli instances generate-client-token my-db

       View credentials for a service instance:
              goli instances show-credentials my-db

       Unbind a service instance from an application:
              goli instances unbind my-db my-app

SEE ALSO

run-query

NAME
       goli run-query - Run a direct query on the database

SYNOPSIS
       goli team-features run-query [OPTIONS]
       goli team-features query [OPTIONS]

DESCRIPTION
       The goli run-query command, or its alias goli query, allows you to run a direct SQL query on the database
       without needing to open or manually interact with the database.
       It executes SQL queries (SELECT queries only) directly from the CLI, enabling users to retrieve data in
       the database.

OPTIONS
       -h, --help
              Display this help message and exit.

       -q, --query
              The SQL query you wish to execute on the database.
              If not specified, interactive mode will open to input the query.

       -d, --db
              The name of the database where the query will be executed.
              If not specified, interactive mode will open to choose the database.

       -f, --file
              The name of the file to save the query result to.
              If not specified, the query result will be displayed in the terminal.

EXAMPLES
       Execute the query "SELECT * FROM users" on the "my_database" database:
              goli run-query -q "SELECT * FROM users" -d "my_database"

       Execute the query "SELECT * FROM users" and save the result to the "users.csv" file:
              goli run-query -q "SELECT * FROM users" -f "users.csv"

       Open interactive mode to input the query and select the database:
              goli run-query

INTERACTIVE MODE
       If the query or database is not specified, the command will enter interactive mode, allowing the user to
       input the SQL query and select the database from the available options.

SEE ALSO
         team-features, connect-to-db, postgres

    • NOTE: If you see a security warning that the file is damaged,

    • After downloading the zip file, unzip it by right-clicking and selecting Extract All.

Setup Instructions:

  • For macOS:
    • 1. Move the file to the wanted directory.
    • 2. Open your .zshrc file and paste this in the file(make sure to replace #path# with the actual path):
      autoload -U compinit; compinit
      source "#path#"
      compdef _goli goli
    • 3. You should now have auto-completion enabled for the goli CLI in your terminal.
  • For Windows:
    • 1. Move the file to the appropriate directory, for example: C:\Program Files\goli.
    • 2. Open PowerShell and run: Set-ExecutionPolicy RemoteSigned to allow script execution.
    • 3. Run: source ~/.bash_profile or source ~/.zshrc to apply the changes.
    • 4. You should now have auto-completion enabled for the goli CLI in your terminal.
